Optimizing the Vizio D40u-D1 TV Video and Audio Settings

by
Nicholas F. Palomba
on March 26, 2025


I purchased a Vizio D40u-D1 40-inch TV from Amazon in 2016 to upgrade my viewing experience right before the release of UNCHARTED 4: A Thief's End. Although this model can be considered ancient technology and I do intend to upgrade soon, in this article I list the video and audio settings that I use and attempt to justify their utilization. Presently, I use the TV with my PlayStation 3, PlayStation 5, Nintendo Switch, and some old game consoles with analog video. Therefore, I have optimized the TV settings for gaming. The review and settings from RTINGS.com were helpful for the development of this list. Additionally, I have used the read-edid and edid-decode software packages in an attempt to further investigate the capabilities of the TV.

Setting Evaluation

Firstly, I made custom picture modes via the TV's interface to adjust the picture settings and save my choices. There are, however, several default picture modes available. Game mode is claimed to reduce throughput delays and optimize the picture settings for displaying game console output. Computer mode is claimed to optimize the picture settings for displaying computer output. Since there are possibly hidden effects made by the picture modes that cannot be adjusted by the user, it is probably best to use the game or computer picture mode, rather than another picture mode available, as the base for the custom picture mode created. Most often, game consoles and computers produce video and audio signals that are never compressed by a lossy algorithm, particularly when playing video games. It is less likely that these two picture modes apply hidden effects that process the signal and attempt to improve it, while actually altering the signal from that intended by the game designers and increasing lag. These hidden effects, if any, cannot add detail that does not exist. This reasoning forms the basis for most of the chosen settings in this article.

Surprisingly, the game picture mode does not have the clear action and reduce block noise settings off by default. Similarly, the computer picture mode does not have the game low latency setting on and the film mode setting off by default. Because of this, I chose the lesser of two evils and selected the game picture mode as the base for the two custom picture modes that I created. As seen below, only one setting differs between the two picture modes created.

Auto Brightness Control

The auto brightness control setting detects the light level in the room and automatically adjusts the backlight. This is set off, which ensures a consistent output.

Backlight

The backlight setting adjusts the backlight brightness, which does not directly affect the black or white levels and is essentially the true brightness setting. In Fig. 1, I have the backlight set to the maximum available setting to minimize the effects of reflections from other light sources in the room. This selection should be based on the lighting conditions and be set as desired.

Brightness, Contrast, Color, Tint, Gamma, and Color Tuner

The brightness (black level), contrast (white level), color (color intensity), tint (hue), gamma (gamma curve), and color tuner settings are best set with a screen calibration device, which I intend to eventually do. Each TV, even those of the same model, must be set individually and likely differently to obtain the optimal color results. As of this writing, I have left the settings at their default values, which subjectively appear acceptable to me. The default values for both the game and computer picture modes are as follows: brightness is 50, contrast is 50, and color is 63. The default tint setting is 0 for all of the available picture modes. The default gamma setting is 2.2 and 2.1 for the game and computer picture modes, respectively. This has been left at the setting of 2.2.

Sharpness

The sharpness setting adjusts the edge sharpness of picture elements. It can be used to sharpen non-4K content. However, it cannot produce detail that does not otherwise exist in the source signal. Unfortunately, I do not know for sure which setting (e.g., a default or zero value) leaves the source signal unaltered. The default value for the game picture mode is 25 and for the computer picture mode is 0. I presume that the setting of 0 applies no sharpness filter and have set it as such. When playing 1080p or lower resolution games, I may play with this setting to subjectively test values that sharpen edges without creating noticeable artifacts.

Color Temperature

The color temperature setting is used to select a preset color temperature. Normal is claimed to be optimized for television viewing. Cool produces a blue-hued picture. Computer is claimed to optimize the picture for use as a computer monitor. Both the game and computer picture modes have this set to computer by default. This is set to computer, since again this option seems most likely to leave the displayed video unaltered from the source signal.

Black Detail

The black detail setting is used to adjust the average brightness of the picture to compensate for large areas of brightness. This can be set to off, low, medium, or high. This is claimed to improve details in dark areas of the picture, and high provides the strongest effect. Both the game and computer picture modes have this set off by default. This is set off, because it would otherwise alter the displayed video from the source signal.

Active LED Zones

The active LED zones setting dynamically improves the contrast ratio of the picture by adjusting the backlight of eight separate zones individually. The adjustment is controlled on the fly based on the content displayed on the screen. Both the game and computer picture modes have this set off by default. This is set on. While this enhances contrast, haloing effects around bright objects in dark scenes are a common side effect of local dimming that one might want to be on the lookout for. I have had this setting on for years, and it has not bothered me.

Reduce Judder

The reduce judder setting activates motion compensation, which suppresses stuttering of the image for low-frequency content when the camera moves across a scene. As the setting increases, judder is reduced. This is set to a value of 5 for the usual reason, which is the default for all of the available picture modes. Although I cannot be certain, presumably this value applies little or no processing of the source signal. When playing low frame rate games on the old game consoles with analog video, I may experiment with this setting if I experience stuttering.

Clear Action

The clear action setting reduces blur in scenes with fast action but limits the range for the backlight setting. This is set off. It would otherwise alter the displayed video from the source signal.

Reduce Noise

The reduce noise setting opens a sub-menu with two settings that are both set off. The reduce signal noise setting diminishes artifacts in the image caused by the digitizing of motion images and is settable to off, low, medium, or high. This only affects analog video sources, and I may experiment with this setting if I experience speckle noise when playing games on the old consoles with analog video. The reduce block noise setting reduces pixelation and distortion for mpeg files and can be set to off, low, medium, or high.

Game Low Latency

The game low latency setting is set on to reduce the lag between the source signal and the displayed video to a minimum.

Film Mode

The film mode setting is claimed to optimize the picture for watching film and is settable to auto or off. It appears to be forced off with the game low latency setting on and should be set off anyway.

Color Space

The color space setting is used to match the TV's color space to that of the source signal and can be set to auto, YCbCr, or RGB. TV video sources use YCbCr, but computers use RGB. The PlayStation 3, PlayStation 5, and Nintendo Switch all produce RGB video when gaming.

Unfortunately, the TV does not support RGB video for a resolution of 4K at 60 Hz, and therefore, the PlayStation 5 must output compressed YCbCr video when operating at this resolution and frequency. This also implies that the full RGB range cannot be supported for 4K, 60 Hz content. As such, this required me to make a custom picture mode specifically for the PlayStation 5 with the color space set to auto. Furthermore, the color space setting is set to auto for the analog TV inputs and cannot be changed by user.

The color space setting is set to RGB for the other custom picture mode that I created for the PlayStation 3 and Nintendo Switch, which are both limited to a maximum resolution of 1080p at 60 Hz. The auto setting appears as a tempting selection, but I don't suspect it to be reliable, particularly when it comes to choosing between the limited and full RGB range (supposing the TV does). Since the EDID evaluation below doesn't conclusively verify that the TV does support the full RGB range, the TV must be tested to confirm that it supports the full range during the needed calibration mentioned above. However, all of the computer-oriented support mentioned throughout this article strongly suggests that the TV does support the full RGB range, which makes the selected setting ideal. It is the option that is most likely to pressure the TV to operate properly based on the source signal with the full RGB range.

Surround Sound

The DTS Studio Sound setting allows the use of proprietary technology to emulate surround sound with the two-speaker playback system, including volume leveling and bass enhancement. Since this alters the output sound from the source signal, it is set off.

Volume Leveling

The DTS TruVolume setting allows the use of proprietary technology to provide volume leveling, or consistent volume levels during transitions between content, content formats, and input sources. Since this alters the output sound from the source signal, it is set off.

Display Mode

The TV can display images in five different modes: normal, stretch, panoramic, wide, and zoom. The normal setting preserves the content's original aspect ratio and size. Panoramic and wide can be used for old games with a 4:3 aspect ratio to remove the black bars on each side by stretching the content to fill the screen, but this distortion of the image will be evident.

Viewing Data

The viewing data setting should be set off, especially if the TV is connected to the internet. This prevents Vizio from tracking what is viewed (to a degree, at least). Generally, I don't connect my "smart" TVs to the internet.

HDMI Port Selection

HDMI ports 1, 2, 3, and 4 support 4K input at up to 30 Hz. The manual states that HDCP 2.2 is supported on HDMI ports 1, 2, and 5. HDMI port 5 supports 4K input at up to 60 Hz. Although, as alluded to above, the color format is unfortunately limited to YUV420 at 4K, 60 Hz. This port is used for my PlayStation 5. For best performance with HDMI port 5, only devices that are capable of outputting 1080p or 4K resolution should be connected (e.g., the reduce signal noise setting is not available to be customized for port 5). HDMI port 1 supports audio output (ARC) and is the TV's highest quality audio output. HDMI port 2 is selected for use with the PlayStation 3 and Nintendo Switch, or more specifically, it is used for the loop-through output of my HD capture card.

EDID Evaluation

EDID information is metadata provided by displays to describe their capabilities to the connected video source device. To further investigate the capabilities of the TV, the decoded EDID information of each HDMI input is provided below in Fig. 3 - 7. After connecting my laptop to each of the five HDMI inputs and saving the binary EDID information with get-edid, the command in Fig. 2 was used on each binary file to perform the decoding and save the information in a text file. The serial number of the TV has been manually zeroed out in the hex section of the decoded EDID information provided.

The EDID information seems to be shoddily developed or incorrectly configured, particularly the identical information for HDMI ports 3 and 4. The information appears generic and inconsistent. This makes the data hard to trust, but I have proceeded with the data as presented.

Sadly, the EDID information for each port does not confirm or deny support for the full RGB range via a flag. However, the DC_30bit output for HDMI ports 1, 2, and 5 seems to confirm support for 30-bit color depth (10 bits per color channel) as specified on Vizio's website for this TV. The information for HDMI ports 3 and 4 indicates that only 8 bits per color channel are supported. As such, it's probably best to avoid those two inputs in the rare case that the source does support 10-bit color depth. It is certain, however, that the TV's lack of HDR supports means that the 10-bit color support isn't utilized in this modern context.
